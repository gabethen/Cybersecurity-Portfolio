# Mission 09 - Packet Analysis

## Objective

Learn how SOC analysts examine network packets to determine whether activity is normal or potentially malicious.

---

## Lesson Summary

When devices communicate across a network, data is divided into packets. Each packet contains valuable information that helps security analysts understand what is happening on the network.

Key packet information includes:

- Source IP – The device sending the traffic.
- Destination IP – The device receiving the traffic.
- Protocol – The communication method used (TCP, UDP, ICMP).
- Port – The service or application being accessed.
- Packet Size – The amount of data being transferred.

SOC analysts review packet information to identify suspicious activity such as unauthorized remote access, unusual communication patterns, or possible data exfiltration.

---

## Scenario

While monitoring network traffic, you observe the following information:

| Field | Value |
|--------|-------|
| Source IP | 192.168.1.45 |
| Destination IP | 203.0.113.77 |
| Protocol | TCP |
| Destination Port | 22 |
| Time | 3:18 AM |
| Packet Size | 1,850 bytes |

Additional information:

- The employee normally works from **9:00 AM to 5:00 PM**.
- The destination IP is **not recognized** as one of the company's normal servers.
- There were **150 similar connections** within the last 10 minutes.

---

# Questions & Answers

## 1. What service is commonly associated with port 22?

SSH (Secure Shell) is commonly associated with port 22. It is used to securely connect to and manage remote systems over a network.

---

## 2. Which details in this scenario could be considered unusual?

Several details stand out as unusual:

- The connection occurred at **3:18 AM**, outside the employee's normal working hours.
- The destination IP address is unknown.
- There were **150 similar connections** in only 10 minutes.
- SSH traffic may be unexpected if the employee does not normally perform remote administration.

---

## 3. Does this information prove malicious activity? Why or why not?

No. Although the activity is suspicious, it does not prove malicious behavior. It could be caused by a scheduled task, an authorized administrator, or another legitimate process. Additional evidence is required before making a conclusion.

---

## 4. What would you investigate next?

I would investigate:

- Whether the employee was logged in at 3:18 AM.
- The owner and reputation of the destination IP address.
- Whether similar SSH connections have occurred previously.
- Windows login and authentication logs.
- Firewall and SIEM logs for related events.
- Whether any files or large amounts of data were transferred.
- Whether any alerts were generated by endpoint security software.

---

# What I Learned

I learned that packet analysis allows SOC analysts to investigate network activity by examining packet details such as IP addresses, ports, protocols, timing, and traffic patterns. Instead of assuming activity is malicious, analysts gather evidence from multiple sources before making a conclusion.
